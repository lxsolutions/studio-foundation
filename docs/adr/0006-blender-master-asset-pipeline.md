# ADR 0006: Blender master-asset pipeline

- Status: Accepted
- Date: 2026-07-19

## Context

Characters, environments, rigs, animation, collision proxies, LODs, destruction
variants, cinematics, and procedural assets need one master source and a deterministic
path into the engine, automatable by CI and AI agents.

## Decision

- **Blender** (pinned LTS; currently 5.2.0 LTS) is the master source. `.blend` files in
  `assets-source/` are the only editable asset truth.
- All automation uses **Blender background mode + Python API** (`blender -b --python`)
  through `tools/asset-pipeline/` — no GUI-only automation, no arbitrary remote code
  execution into Blender (community "Blender MCP" servers stay disabled by default;
  only our whitelisted scripts are supported — ADR 0009).
- Interchange/runtime: **glTF/GLB** for scenes/meshes/skins/animation; KTX2/Basis and
  platform texture compression at cook time where beneficial; WAV/lossless audio
  sources cooked to runtime formats.
- Every asset has a **JSON sidecar** (`<name>.meta.json`, schema in
  `shared/schemas/asset-meta.schema.json`): id, license, source, creator, provenance
  (including AI-generation fields), game compatibility, category, LOD/collision/texture
  policies, animation set, platform overrides, performance budget.
- `assets-source/` (masters + sidecars, committed) is strictly separated from
  `assets-generated/` (exports, cooked outputs, previews, reports — gitignored,
  reproducible, hash-cached). Generated outputs never overwrite masters; the pre-commit
  hook rejects direct edits to generated directories.
- Validation (`just asset-validate`) enforces: naming, units/scale, applied transforms,
  origins, UVs, material count, triangle budgets, skeleton/bone conventions, animation
  naming, collision meshes, required LODs, missing textures, unsupported shader
  features, and licensing/provenance metadata.

### Licensing note

Blender is **GPL-2.0-or-later** — used strictly as a standalone tool. Its output
(.blend/GLB content we author) is not GPL-derived; we link nothing against Blender.
This is the documented, isolated GPL usage permitted by our license policy (ADR 0013).

## Consequences

- Artists keep full Blender freedom; the pipeline (not the artist) guarantees engine
  readiness, and failure messages are written for artists and agents alike.
- Determinism + source-hash caching make asset cooking CI-friendly and incremental.
- The bootstrap sample asset (`crate_a`) is generated by a whitelisted script and
  committed as a normal master + sidecar, proving the chain end to end.
