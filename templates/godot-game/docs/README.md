# Game project (generated from studio-foundation)

| Piece | Where | Notes |
|---|---|---|
| Godot project | `project/` | Opens in the pinned official editor; `Studio` autoload boots studio_core |
| Shared addon | `project/addons/studio_core/` | **Generated copy** — edit `shared/godot-addons/` in the foundation, then `just godot-sync-addons` |
| Master assets | `assets-source/` | `.blend` + `.meta.json` sidecars (ADR 0006) |
| Cooked assets | `project/assets/generated/` | Generated by `just asset-cook` — never edit |
| Game server | `server/` | Standalone crate reusing foundation crates by relative path; owns `game_<id>` DB schema in `server/migrations/` |
| Tests | `project/tests/` + `server/tests/` | `just test-godot GAME=<this>` / `cargo test` in `server/` |

## Consumption models (ADR 0015)

1. **Early development (this setup):** relative-path references to the shared
   platform inside the monorepo. Zero-friction iteration on shared code.
2. **Stable games:** pin versioned releases of `studio-protocol`/
   `studio-dedicated-server` (git tags or a registry) and a released
   `studio_core` addon; upgrade deliberately.

Switching is a dependency edit, not a restructure — nothing in game code may
depend on the foundation repo's absolute filesystem location.
