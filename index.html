Enter the colosseum Source Minimal scene
First load takes a while — the page will tell you where it is. It downloads ~45 MB of engine (12 MB compressed) and then compiles every shader to WGSL for your GPU. On a ~12 Mbps connection that measured about 13 seconds end to end, most of it the download; a slower link takes proportionally longer. A loading screen stays up until frames are actually being drawn, so you won't be staring at a black canvas wondering if it broke.

What you are actually looking at

  • Official Godot 4.7.1 (MIT), unmodified as the base — pinned by full commit hash.
  • A WebGPU export backend maintained as an ordered, SHA-256-locked patch series (currently 14 patches), each one auditable in the repo.
  • Rendering through the Forward Mobile renderer with real materials, lighting, and shadows — not a canvas trick, and not WebGL 2.
  • Verified on real hardware (an NVIDIA Tesla P40): locked 60 fps at roughly 490–630 draw calls and ~23M primitives per frame, with 0 GPUValidationError.

Honest status: WebGPU support is beta. Some post-processing effects are still disabled under WebGPU, and WebGL 2 remains the supported fallback for production.

Reproduce it yourself

The engine, the patch series, and the minimal scene are checksum-locked and rebuildable from the repository. The Chariot Club's game content is not published — it is a studio game, so the colosseum above cannot be rebuilt from source today. The scene at /showcase/ is the one you can reproduce end to end, and it exercises the same render path. 

git clone https://github.com/lxsolutions/studio-foundation.git
cd studio-foundation
just engine-fetch      # official Godot + verified patch series
just engine-build      # build the WebGPU export templates
just export-browser-webgpu

Skipping the multi-hour engine build: prebuilt export templates are published with their SHA-256 hashes.

Why this project exists

Godot does not accept AI-generated contributions and has said it does not intend to add AI features to the engine. That leaves an open lane, and Studio Foundation takes it: an AI-native, open-source game-dev toolkit — MCP server, agent workflows, and an AI-assisted engine backend — built in the open and reproducible by construction, so "AI-built" never means "unauditable."

Official Godot stays the upstream. We own the distribution, not the engine.
