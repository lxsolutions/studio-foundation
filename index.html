What you are actually looking at

  • Official Godot 4.7.1 (MIT), unmodified as the base — pinned by full commit hash.
  • A WebGPU export backend maintained as an ordered, SHA-256-locked patch series (currently 13 patches), each one auditable in the repo.
  • Rendering through the Forward Mobile renderer with real materials and lighting — not a canvas trick, not WebGL 2.
  • Verified on real hardware (an NVIDIA Tesla P40) before shipping, not just "it compiled."

Honest status: WebGPU support is beta. Some post-processing effects are still disabled under WebGPU, and WebGL 2 remains the supported fallback for production.

Reproduce it yourself

Everything here is checksum-locked, so you can rebuild and re-verify it independently.

git clone https://github.com/lxsolutions/studio-foundation.git
cd studio-foundation
just engine-fetch      # official Godot + verified patch series
just engine-build      # build the WebGPU export templates
just export-browser-webgpu

Why this project exists

Godot does not accept AI-generated contributions and has said it does not intend to add AI features to the engine. That leaves an open lane, and Studio Foundation takes it: an AI-native, open-source game-dev toolkit — MCP server, agent workflows, and an AI-assisted engine backend — built in the open and reproducible by construction, so "AI-built" never means "unauditable."

Official Godot stays the upstream. We own the distribution, not the engine.
